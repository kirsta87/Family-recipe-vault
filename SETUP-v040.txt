# Recipe Vault v0.40 — Structured HelloFresh importer

This update imports the recipe page itself, not just the URL.

It attempts to store:

- name
- image
- protein
- meal type
- cuisine
- tags
- prep/cook/total time
- ingredients
- instructions
- nutrition

The importer first looks for Recipe JSON-LD, then uses a HelloFresh page-text fallback.

## Before installing

Your sheet must include these final headers after `last_made`:

- ingredients
- instructions
- nutrition

## 1. Update Apps Script

Open `APPS-SCRIPT-v040-IMPORTER.txt`.

Replace:

`KEEP-YOUR-EXISTING-FAMILY-KEY-HERE`

with the exact key already used by your site.

Copy the entire script into Apps Script, replacing the old code.

Save, then:

1. Deploy
2. Manage deployments
3. Pencil icon
4. New version
5. Deploy

The web-app URL stays the same.

## 2. Update GitHub

Replace/add:

- index.html
- app-v040.js
- styles-v040.css

Do not delete the older files yet. The new index only uses v040.

In GitHub Desktop:

- Commit message: `Add structured HelloFresh importer`
- Commit to main
- Push origin

## 3. Test carefully

Use a HelloFresh recipe that is not already in the sheet.

Paste its URL into Add recipe.

After importing, verify:

- the recipe appears
- ingredients and instructions appear inside the recipe window
- the sheet contains JSON text in ingredients and instructions
- ratings still save
- notes still save

If the importer reports that structured details could not be extracted, do not bulk import yet. The parser will need adjustment for that page format.
